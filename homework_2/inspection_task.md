## 1. Общие замечания по документу

### 1.1. Название документа
**Текущее:** «АКТ ОБСЛЕДОВАНИЯ ОБЪЕКТА»

**Предлагаю:** «Акт обследования объекта для внедрения системы интеллектуальной видеоаналитики безопасности»

*Обоснование: более формально и понятно, сразу отражает назначение документа.*

---

### 1.2. Уточнение названий объектов
Объекты №1,2,3,4 названы одинаково («Крючкогиб Suzhou Hanxin...»). При заполнении возникнет путаца.

**Предлагаю:** переименовать в:
- №1: Крючкогиб Suzhou Hanxin (№1)
- №2: Крючкогиб Suzhou Hanxin (№2)
- №3: Крючкогиб Suzhou Hanxin (№3)
- №4: Крючкогиб Suzhou Hanxin (№4)

---

### 1.3. Идентификация объектов
В начале блока каждого объекта отсутствует чёткая привязка к ТЗ.

**Предлагаю:** добавить перед «Геометрией установки камеры»:

> **Номер объекта по ТЗ:** ______
> **Точное наименование станка / зоны:** ______________________

---

### 1.4. Общий вывод по объекту
После заполнения всех параметров нет места для итогового вердикта.

**Предлагаю:** добавить в конец блока каждого объекта:

> **Общий вывод по объекту:**
> - [ ] Подходит текущая камера
> - [ ] Требуется замена камеры
> - [ ] Нужна новая камера
> - [ ] Требуется дополнительная камера
>
> **Комментарий:** ___________________________________

## 4. Сводный чек-лист перед выездом (памятка)

Перед выездом на объект рекомендую убедиться, что:

- [ ] Распечатан акт с запасными копиями (по числу объектов)
- [ ] Подготовлены: люксметр, рулетка/дальномер, камера, маркер
- [ ] У заказчика заранее запрошены: планировка DWG/PDF, схема сети, документация на станки
- [ ] Согласована возможность съёмки спецсценариев с сотрудниками
- [ ] Уточнены требования по категории безопасности (PL/SIL)
- [ ] Включены поля для записи свободных PCIe слотов и ИБП
- [ ] Включена проверка вибраций и электромагнитных помех

---

**После внесения указанных правок документ будет готов к использованию при выезде на объект и полностью пригоден для включения в проектную документацию.**
