# Update ICU

## 1. Инфраструктура и развертывание (миграция с Docker → K8s + Vault)

### Что сделали/делаем
- Переходим с Docker Compose + Docker Secrets на Kubernetes + HashiCorp Vault.
- Уже подняли Kubernetes кластер, тестируем deployment'ы.
- В процессе работы с Vault для внедрения и настройки политики доступа к секретам. 

### Преимущества перехода
- Масштабируемость и оркестрация: K8s сам масштабирует под нагрузку (в том числе под GPU - что критично для наших), делает rolling updates, self-healing и service discovery — в Compose такого «из коробки» нет.
- Безопасность секретов: Vault даёт динамические credentials (они живут минуты и ротируются автоматически), шифрование на лету, доступ между namespace'ами и даже за пределами кластера. Docker Secrets — это статические файлы, которые могут утечь (если развертывание происходит в облаке) и не масштабируются.
- Прод-готовность: проще CI/CD через GitOps, мульти-окружения (dev/stage/prod), горизонтальное масштабирование под сотни камер и событий.

### Куда идем 
Полностью облако-независимый, отказоустойчивый прод.

## 2. UX/UI: новый дизайн-макет приложения

### Что делаем
Полностью перерабатываем интерфейс. Старый UX был основан на парадигме «video streams first» — пользователь тонет в десятках открытых стримов, что давно стало узким местом: визуальная перегрузка, плохая масштабируемость интерфейса. Переходим на «AI / events first»: в центре — аналитика, события, объекты.

### Примеры приложений «AI/events first» 

**В разработке:** 
- [Cursor](https://cursor.com/product): открываешь - сразу чат; агент пишет код в реальном времени, минимизируя участие человека; рассчитан на <u>массового</u> разработчика.
- [Kiro](https://kiro.dev/): схожий с предыдущим, но под большим контролем: кидаешь PRD → AI генерит план, задачи выполняет с разрешения.

**В видеоаналитике:**
- [Coram AI](https://www.coram.ai/security-camera-system): чистый AI-first security: не галерея стримов, а события с приоритетами.
- [Lumana AI](https://www.lumana.ai/products/visual-intelligence): та же концепция, но плюсом - автоматическая генерация подписей к событиям.
- [IRIS (Tata Elxsi)](https://www.tataelxsi.com/platforms/iris-ai-driven-video-analytics): промышленная платформа, в центре которых события.
- [Vaidio AI + WiseSeek AI NVR](https://www.vaidio.ai/platform): та же концепция, но плюсом - поиск по архивам (над чем мы и работаем).

То есть данная парадигма - уже стандарт для массового пользователя видеоаналитики.

### Ключевые изменения
- Человек физически не может уследить за десятками (даже не сотнями) камерами → акцент на автоматизацию и умные события.
- Меньше модальных окон → полноэкранные дашборды (особенно критично для настройки детекторов и планов помещений с множеством этажей, помещений, камер).
- Шрифты, иконки, цвета — уменьшаем, чтобы не рябило в глазах.
- Перерабатываем уведомлений (чтобы не перекрывали интерфейс).
- Убираем пустое пространство (сейчас его тонны) и убираем лишние элементы, с которыми почти не взаимодействуют.
- Списки сотрудников, номеров авто, камер и пр. — с нормальными фильтрами, поиском и группировкой.


### Куда идем
Современный, расширяемый интерфейс, который легко допиливать под новые модули аналитики.

## 3. Новые фичи: поиск по архивам (видео + аудио)

### Что делаем (видео-часть)
- При изменении сцены не просто стартуем/стопаем запись, а автоматически детектим и векторизуем ключевые объекты (люди, авто и т.д.).
- Стек: переработанный (легковесный -> ускоренный) детектор для вычленения объектов в кадре + модель для векторизации объектов + уже работающая векторная БД.
- Результат: поиск по изображению или текстовому запросу + мгновенная выдача таймкодов и отрезков видео.

### Что делаем (аудио-часть)
- Параллельно — прорабатываем архивацию по аудиопотоку: при звуковых скачках захватываем фрагмент → векторизуем через подобную модель выше, но работающую с аудио.
- Поиск по примитивам («крик», «лай собаки», «выстрел» и т.п.).

### Куда идем
«Умный» архив, где можно искать не по времени, а по смыслу. Это killer-feature для всех заказчиков.

## 4. Улучшения текущих сервисов

### Что делаем
- Тестирование + фикс багов по всему стеку.
- Критическая правка со стримами: вместо загрузки больших .mp4 из MinIO (объектное хранилище с шифрованием) переходим на HLS-стриминг (HTTP Live Streaming).
- Проблема была в долгой подгрузке тяжелых объектов — HLS решает это полностью (chunk'и + adaptive bitrate).

### Куда идем
Стабильный, быстрый продукт без «тормозов» на проде.

## 6. Инфраструктурные риски: зеркало Git

### Ситуация
- Кода уже очень много. Если с текущим сервером (в офисе 204) что-то случится — потеряем всё.
- Облачные отечественные (так как иностранные под запретом) решения (GitFlic, GitVerse и т.п.) — дорогие для наших объемов.

### Предложение
- Выделить физический компьютер со склада под зеркало Git (можно маломощный).
- Настроить ежедневный git mirror + репликацию метаданных.
- Это обеспечит автономное резервирование без затрат.

### Вопрос 
Можно ли выделить компьютер со склада? Это самый дешевый и надежный вариант.
